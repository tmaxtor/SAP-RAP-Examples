🔒
Lock Propagation
El lock del Root se propaga automáticamente hacia los nodos hijo. Los nodos L2 y L3 usan lock dependent.
🔑
UUID como clave
Se usan sysuuid_x16 para las claves primarias y FK, garantizando unicidad sin necesidad de rangos de numeración.
⚡
Managed Implementation
El framework gestiona automáticamente CRUD y la numeración UUID. Solo se codifica lógica de negocio específica.
🔁
ETag para optimistic lock
Cada tabla incluye local_last_changed_at como campo ETag para control de concurrencia optimista.

managed implementation in class ZBP_I_SALESORDER unique;
strict ( 2 );

lock master
authorization master ( instance )
etag master LocalLastChangedAt
{

  mapping for zsales_order {

ZBP_I_SALESORDER — Behavior Pool
// Handler Class · confirmOrder action · validateCustomer validation
ABAP ZBP_I_SALESORDER.clas.abap 
CLASS zbp_i_salesorder DEFINITION PUBLIC ABSTRACT FINAL
  FOR BEHAVIOR OF zi_salesorder.
ENDCLASS.
CLASS zbp_i_salesorder IMPLEMENTATION.
ENDCLASS.

"--- Handler Class ---
CLASS lhc_salesorder DEFINITION INHERITING FROM cl_abap_behavior_handler.
  PRIVATE SECTION.
    METHODS:
      confirmOrder FOR MODIFY
        IMPORTING keys FOR ACTION salesorder~confirmOrder RESULT result,
      validateCustomer FOR VALIDATE ON SAVE
        IMPORTING keys FOR salesorder~validateCustomer.
ENDCLASS.

CLASS lhc_salesorder IMPLEMENTATION.

  METHOD confirmOrder.
    MODIFY ENTITIES OF zi_salesorder IN LOCAL MODE
      ENTITY salesorder
        UPDATE FIELDS ( Status )
        WITH VALUE #( FOR key IN keys
                        ( %tky   = key-%tky
                          Status = 'CO' ) ).
    READ ENTITIES OF zi_salesorder IN LOCAL MODE
      ENTITY salesorder ALL FIELDS WITH
        CORRESPONDING #( keys )
      RESULT DATA(orders).
    result = VALUE #( FOR order IN orders
                        ( %tky   = order-%tky
                          %param = order ) ).
  ENDMETHOD.

  METHOD validateCustomer.
    READ ENTITIES OF zi_salesorder IN LOCAL MODE
      ENTITY salesorder
        FIELDS ( Customer ) WITH CORRESPONDING #( keys )
      RESULT DATA(orders).

    LOOP AT orders INTO DATA(order).
      IF order-Customer IS INITIAL.
        APPEND VALUE #(
          %tky = order-%tky
          %msg = new_message_with_text(
                   severity = if_abap_behv_message=>severity-error
                   text     = 'Customer is mandatory' )
        ) TO reported-salesorder.
      ENDIF.
    ENDLOOP.
  ENDMETHOD.

ENDCLASS.
